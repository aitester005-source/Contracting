# Facebook Feed Integration Guide

This document provides instructions for managing and configuring the Facebook Feed integrated into the Hedefy Scaffolding website.

## 1. Technical Overview
- **Component**: `components/FacebookFeed.tsx`
- **Styles**: `styles/FacebookFeed.module.css`
- **Method**: Official Facebook Page Plugin (XFBML) via Facebook SDK v18.0.
- **Pages**: Integrated into Homepage (`pages/index.tsx`) and Contact Page (`pages/contact.tsx`).

## 2. Configuration
The component is pre-configured with the provided Facebook Profile ID. To change the target page, update the `pageUrl` prop in the component usage or the default value in `FacebookFeed.tsx`:

```tsx
// Example of custom URL usage
<FacebookFeed pageUrl="https://www.facebook.com/YourNewPageURL" />
```

## 3. Facebook App ID (Optional but Recommended)
For production environments, it is recommended to use your own Facebook App ID to avoid rate limits and enable better analytics.

1. Go to the [Facebook Developers Portal](https://developers.facebook.com/).
2. Create a new App (Type: Consumer or Business).
3. Copy your **App ID**.
4. Update the SDK script URL in `FacebookFeed.tsx`:

```tsx
<Script 
  src="https://connect.facebook.net/en_US/sdk.js#xfbml=1&version=v18.0&appId=YOUR_APP_ID" 
  strategy="afterInteractive"
  ...
/>
```

## 4. Troubleshooting
- **Feed not loading**: Ensure the Facebook Page is public and has no age or country restrictions.
- **Plugin doesn't show posts**: Some Personal Profiles (even in Professional Mode) may have privacy settings that prevent the Page Plugin from rendering. If this happens, ensure the account is converted to a standard Facebook Page.
- **Mobile View**: The feed is set to `data-adapt-container-width="true"`, meaning it will automatically adjust to the width of its parent container on mobile devices.

## 5. Hosting in cPanel
If you are deploying to a cPanel environment:
1. Run `npm run build` followed by `npm run export`.
2. Upload the contents of the `out` directory to your `public_html` or target subdirectory via File Manager or FTP.
3. The PHP environment will serve the static HTML/JS/CSS files generated by Next.js.
